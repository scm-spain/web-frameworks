Setup

Installing Sass

Sass makes CSS fun again. Sass is an extension of CSS3, adding nested rules, variables, mixins, selector inheritance, and more. It’s translated to well-formatted, standard CSS using the command line tool or a web-framework plugin.
  1. Download the ruby language installer
  2. On install check the Add ruby to your PATH
  3. Open the command line
  4. Check your ruby installation with: > ruby --version
  5. Install sass: > gem install sass

Installing the Visual Studio Tools

  1. Install Web Essentials
  2. Install Mindscape Web Workbench
  3. Configure Mindscape Web Workbench
    Goto Herramientas > Opciones > Web Workbench > Add generated files to project > False
    Goto Herramientas > Opciones > Web Workbench > Disable Sass Compilation > True

Installing LiveReload

LiveReload monitors changes in the file system. As soon as you save a file, it is preprocessed as needed, and the browser is refreshed. Even cooler, when you change a CSS file or an image, the browser is updated instantly without reloading the page.
  1. Install livereload
  2. Install the browser livereload plugins for your browser of choice: Chrome or Firefox:
  3. Open the livereload app
  4. Add a new project
  5. Choose the path of the folder you want livereload to monitor
  6. Select your project and type the url you use for development, ex: http://dev.fotocasa.es
  7. Go to your web browser and enable livereload, the live reload app will update the number of browsers connected at the bottom
  8. Open a sass file within your editor of choice. Make a change. Save the file. See it inmediately on your browser =)

Naming conventions

There are only two hard things in Computer Science: cache invalidation and naming things. -- Phil Karlton

These are *guidelines*, use your judgement when breaking the rules

all selectors should go in *lowercase* 
div .box .filters {color: red}
why?
  • visual consistency: html tags go in lowercase, classes should go to
  • simplicity: .property-title is different from .Property-title, don't make me guess
avoid ids and attributes other than classes on your css selectors
#propertyList {color: red}
.property-list {color: red}
an id is always and unique selector
if your tag is indeed unique use a class prefixed by unique-
.unique-property-list {color: red}
why?
  • conflict avoidance:
    leaving your ids and data attributes for exclusive use on your javascript, frees you from knowing if deleting a certain id will cause changes to the way that element looks
  • avoiding specificity issues:
    ids have greater specifity than classes, avoiding ids altogether will prevent specifity problems
  • modularity:
    id implies uniqueness, avoiding ids can help you think in terms of groups of things rather than "this specific thing"

use meaningful class names avoid class names that specify what they look like in their css rules 

.big-red-box {font-size: 16px; background: red}
.alert {font-size: 16px; background: red} // better

use "searchable" class names

.property-module or .property-unit is better than just .property if you're working on a real estate app a search for "property" will yield thousands of results

don't use "ad", or "ad-" as an id or class name

ad blockers will block your content
Inside each folder there is a meta.scss and a selectors.scss Example:

Components > Box > meta.scss inside this file go the mixins that encapsulate the css rules that represent the visual idea of a box 


@mixin box {
    border: 1px solid black;
    padding: 10px;
}

Components > Box > selectors.scss inside this file we associate that visual representation with a css selector 


@import "meta";

.box {
    @import box;
}
this way we can separate the css rules form the selector, and apply those rules to any selector we see fit.

helpers, normalize and theme

helpers.scss provides mixins provides abstractions for css rules that require vendor prefixes like gradients, encapsulates common styling patterns difficult to accomplish with a simple css rule like clearfix, ellipsis and abs-center.

normalize.scss normalizes (thanks Captain Obvious) browser styling, provinding a clean and even surface to begin styling the elements

theme.scss has the initialized variables that control color and structure. Things like $page-width, $text-size-alpha, $text-color-muted go here
